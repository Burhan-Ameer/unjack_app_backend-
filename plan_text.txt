1. Project Overview
Unjack is an Android focus and app-blocking app targeting Gen Z users with a social accountability angle. This document covers the complete backend implementation вҖ” from tech stack selection through database design, folder structure, API contracts, cloud deployment, and phased delivery plan.

Decision
Choice & Rationale
Framework
FastAPI (Python) вҖ” fastest to ship solo, Pydantic validation built-in, auto Swagger docs
Architecture
Monolith вҖ” single app, multiple routers. Microservices add overhead with no benefit at this scale
Database
PostgreSQL on Azure Flexible Server вҖ” standard SQL, zero vendor lock-in, swap with one env var
Auth
DIY JWT with python-jose + bcrypt вҖ” no third-party dependency, works anywhere, ~50 lines
Hosting
Azure Container Apps вҖ” use $90 credit, Dockerized so move anywhere later
Push
Firebase FCM вҖ” free tier, winner-of-week and streak alerts
Cron
APScheduler inside FastAPI вҖ” Monday winner calculation, no separate service needed

MCRS вҖ” Microservices-Ready Monolith Structure
The app is built as a monolith but with MCRS (Modular Clean Router Structure) вҖ” each feature domain is fully self-contained (router, service, schema, model). When a feature outgrows the monolith, it can be extracted into its own service with minimal refactoring.

7. Winner of Week Logic
APScheduler runs inside the FastAPI process. Every Monday at 00:05 UTC it runs the winner calculation job.

Monday 00:05 UTC     в”Ӯ     в–јFor each user: aggregate app_sessions WHERE blocked_at BETWEEN               last Monday AND yesterday вҶ’ SUM(duration_sec)     в”Ӯ     в–јWrite / upsert weekly_stats rows (user_id, week_start, total_focus_sec)     в”Ӯ     в–јFor each friendship group: rank users by total_focus_sec DESC     в”Ӯ     в–јSet weekly_stats.rank for all users in that group     в”Ӯ     в–јFind rank=1 user per group вҶ’ send FCM push to their device     "You won this week! рҹҸҶ Your focus time beat all your friends."

The app binary never imports Azure SDKs. All cloud config is in environment variables and deployment YAML files. To migrate away from Azure:
Change DATABASE_URL env var to new Postgres host
Push Docker image to new registry
Run container on new platform (Railway, Render, VPS, GKE)
Zero code changes required
